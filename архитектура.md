# Astrea Timeline — Архитектура проекта (v3.1)

Документ описывает текущую архитектуру системы. **Источник истины — код в репозитории.**

## 1. Назначение и обзор

**Astrea Timeline** — веб-приложение для построения натальных карт, расчёта транзитов, лунного календаря и генерации AI-интерпретаций с использованием Swiss Ephemeris.

- **Целевая аудитория**: астрологи-любители, профессионалы, люди интересующиеся личным развитием
- **MVP**: натальные карты + транзиты + AI-интерпретации
- **Growth**: планировщик, CRM для астрологов, RAG-чат

**Стек**: 
- Backend: Python 3.12 + FastAPI
- Frontend: React 18 + Vite + D3.js
- DB: PostgreSQL 16 + Redis
- Deployment: Railway (backend) + Vercel (frontend)

**Монетизация**: Подписочная модель **4 тарифов**:
| Тариф | Цена | Целевая группа |
|---|---|---|
| Free | 0 | Пробующие, нетерпеливые |
| Lite | 790₽/мес | Любопытствующие |
| Pro | 1990₽/мес | Регулярные пользователи |
| Premium | 7990₽/мес | Профессиональные астрологи |

| Ключевая метрика | Реализация |
|---|---|
| Точность расчётов | Swiss Ephemeris (pyswisseph), < 1 угл. сек |
| Стриминг текста | Server-Sent Events (секционный XML) |
| Отказоустойчивость LLM | GPT-4o → DeepSeek → шаблоны |
| Масштабируемость | Celery + Redis кэш |
| Контроль расходов | Daily budget + per-tier rate limits |
| Безопасность | JWT + Google OAuth + bcrypt |
| Email | Resend API + SPF/DKIM/DMARC |

## 3. Технологический стек (детально)

### Backend: Python 3.12 + FastAPI

**Основной стек:**
- **FastAPI 0.109+** — асинхронный веб-фреймворк (ASGI)
  - Автоматическая OpenAPI/Swagger документация
  - Встроенная валидация Pydantic
  - Dependency injection система
  
- **SQLAlchemy 2.0** — ORM (Object-Relational Mapping)
  - Поддержка async/await через sqlalchemy[asyncio]
  - Type hints для всех моделей
  
- **Alembic** — миграции БД
  - Версионирование схемы
  - Auto-upgrade на старте

**Хранилище данных:**
- **PostgreSQL 16** (Railway managed)
  - JSON поля для сложных объектов (planets, aspects)
  - JSONB для быстрого поиска
  - Индексы на user_id, created_at, tier
  
- **Redis** (Railway managed)
  - `transit_cache:{chart_id}` — кэш транзитов (TTL: 30 дней)
  - `geo_cache:{lat},{lon}` — кэш геокодирования (TTL: 90 дней)
  - `interp_cache:{profile_hash}` — кэш интерпретаций (TTL: 7 дней)
  - `rate_limit:{user_id}` — счётчики per-tier лимитов (TTL: 1 день)

**Астрономические расчёты:**
- **pyswisseph** (Swiss Ephemeris Python bindings)
  - Полная точность < 1 угловой секунды
  - Поддержка планет, комет, астероидов
  - Файлы .se1 (эфемериды) хранятся в `/app/ephe`

**Аутентификация & Безопасность:**
- **python-jose** — JWT
  - access token: 15 минут, содержит `user_id`, `tier`, `iat`
  - refresh token: 7 дней, содержит `user_id`, `exp`
  - Ротация: `JWT_SECRET_PREV` для перехода между ключами
  
- **passlib[bcrypt]** — хеширование паролей
  - Cost factor: 12 (медленное, устойчивое к brute force)
  
- **slowapi** — rate limiting
  - Per-tier лимиты (Free: 10 req/min, Pro: 100 req/min)
  - Шифрование IP адресов для anti-fraud

**HTTP & Интеграции:**
- **httpx** — асинхронный HTTP-клиент
  - Используется для OpenAI API, DeepSeek API, Resend API
  - Retry logic + exponential backoff
  
- **pydantic 2.7** — валидация
  - Strict mode для входящих данных
  - Field validators для бизнес-логики
  
- **pytz + timezonefinder** — часовые пояса
  - Автоматическое определение TZ по координатам
  - Преобразование дат в UTC
  
- **geopy** — геокодирование
  - Nominatim (OpenStreetMap)
  - Fallback: встроенный cache

**Фоновые задачи:**
- **Celery + Redis broker**
  - `schedule_retention_emails(user_id)` — цепь из 3 задач (Day 2, 7, 14)
  - `generate_pdf_report(chart_id)` — асинхронная PDF-генерация
  - `recalculate_transits(chart_id)` — периодические пересчёты
  - `send_weekly_digest()` — Cron job через Railway

### Frontend: React 18 + Modern Tooling

**Фреймворк:**
- **React 18.3** — компонентная архитектура с hooks
- **React Router 6.23** — клиентский роутинг
- **TanStack Query v5** (опционально) — управление состоянием сервера

**Сборка & Оптимизация:**
- **Vite 5** — lightning-fast dev server (esbuild)
  - HMR (hot module reload)
  - Code splitting автоматический
  
- **Tailwind CSS 3.4** — utility-first стили
  - Custom theme variables (light/dark)
  - Responsive design встроен

**Визуализация:**
- **D3.js 7.9** — интерактивное SVG колесо зодиака
  - Планеты, дома, аспекты
  - Zoom/pan/drag interactions
  
- **date-fns** — форматирование дат без lodash

**Состояние & Коммуникация:**
- **Context API** — AuthProvider, ThemeProvider
- **fetch API + EventSource** — REST + SSE
- **localStorage** — анонимные карты, флаги (onboarding_seen, theme)

### Инфраструктура

**Контейнеризация:**
- **Docker** — Python 3.12-slim базовый образ
  - Multi-stage build для оптимизации размера

**Production Deployment:**

  - Автоматический деплой при push в main
  - Managed PostgreSQL + Redis
  - Railway Cron для scheduled tasks
  
- **Vercel** — Static hosting для frontend
  - Edge functions (опционально)
  - Automatic HTTPS + CDN

**CI/CD:**
- **GitHub Actions**
  - Python tests (pytest)
  - Linting (flake8, black)
  - Type checking (mypy)
  - Frontend build

## 9. Security Deep Dive

| Угроза | Защита | Реализация |
|---|---|---|
| SQL Injection | SQLAlchemy ORM | Параметризованные запросы |
| CSRF | CORS + SameSite | CORS whitelist, Secure flag |
| XSS | CSP + sanitize | React escaping, no innerHTML |
| Brute Force | Rate limiting + bcrypt | slowapi (3 попытки/5м), cost 12 |
| Token theft | JWT + HttpOnly | Refresh rotation, short expiry (15m) |
| API misuse | Rate limiting | Per-tier: Free 10 req/m, Premium 500 req/m |
| Anti-fraud (Premium) | IP monitoring | Redis IP tracking, 3+ IPs за 30м → logout |
| LLM spend overrun | Daily budget + circuit breaker | $50/day limit → template fallback |
| GDPR compliance | Cascading delete | DELETE user → cascade to all charts/subscriptions |

## 12. Типичный user flow (с деталями)

### День 1: Регистрация
1. Пользователь заходит на https://astreatime.ru
2. Нажимает "Создать натальную карту"
3. Заполняет BirthForm (имя, дата, время, место)
4. `POST /chart/calculate` → натальная карта в браузере (localStorage)
5. Видит AI-интерпретацию (обрывается на 7-м доме/Луне/Сатурне)
6. CTA: "Войдите, чтобы сохранить" → AuthModal
7. Регистрируется через email или Google
8. `POST /chart/save-anonymous` → карта привязывается к аккаунту
9. Email: Welcome с Sun sign + персональный инсайт
10. Tier: Free (по умолчанию)

### День 2: Retention (Celery)
1. Celery задача срабатывает: `send_day2_email(user_id)`
2. Email: "Ваш ближайший активный транзит" + ссылка на Chart page

### День 7: Loss Aversion (только Free)
1. Celery задача: `send_day7_email(user_id)`
2. Email: "Вы пропустили 3 важных транзита" + список + CTA "Upgrade"

### День 14: Coupon Offer (только Free)
1. Celery задача: `send_day14_email(user_id)`
2. Создаёт Stripe Coupon (30% на Lite annual)
3. Email: "Скидка 30% на годовой Lite" (24ч действие, один раз)
4. Проверка: `coupons_sent` таблица для дедупликации

### День 30+: Upgrade
1. Пользователь нажимает "Upgrade to Lite" на странице карты
2. `POST /payments/checkout {tier: "lite"}` → Stripe Checkout session
3. Платит 790₽ (или 7900₽ с 30% скидкой = 5530₽)
4. Stripe webhook: `customer.subscription.created`
5. Backend: обновляет User.tier = "lite"
6. Email: "Спасибо за подписку!" + инструкция

### Месячный цикл (Pro/Premium)
1. Каждый понедельник 09:00 МСК: Railway Cron запускает `/api/v1/internal/weekly-digest`
2. Берёт 3 ближайших транзита на неделю
3. Текущую лунную фазу + рекомендации
4. Отправляет email через Resend
